int32_t obis_fixed_point_to_thousandths ( obis_fixed_point_t fixed_point)
static

Convert OBIS fixed-point value to integer (scale by 1/1000)

Converts OBIS fixed-point representation to integer by scaling down by factor 1000. Effectively converts base units to "one thousandth-s" or removes the decimal point for values with 3 decimal places.

OBIS format: whole + frac (6 decimal precision, stored as 1e-6) Conversion: whole * 1000 + frac / 1000 = value scaled by 1/1000

Examples:

  • 123.456789 kWh 123456 Wh
  • 5.678123 kW 5678 W
  • 0.045 m 45 dm
Parameters
fixed_pointOBIS fixed-point value with 6 decimal precision
Returns
int32_t Integer value scaled by 1/1000 (no FPU needed)

Publish P1 data with rolling statistics to P_ONE_CHAN.

Called on DATA_PUBLISH_REQUEST_CHAN event. Only publishes if at least one valid P1 sample was collected during the window. Uses p1_stats.sample_count as the authoritative indicator of data availability.

If sample_count is 0, all acquisitions failed in this window and no ZBUS message is published, preventing stale/empty data from being sent.

Converts OBIS fixed-point values to int32_t Wh/W (no FPU needed!). Uses P1 meter's own epoch timestamp from OBIS 0-0:1.0.0.

void update_rolling_stats ( void )
static

Update rolling statistics with new P1 measurement.

Uses instantaneous power from OBIS 1-0:1.7.0 (delivered/consumed) and 1-0:2.7.0 (received/produced).

Tracks consumed and delivered separately for peak shaving and DEV analysis. Called after each successful P1 data acquisition and parsing. Uses int32_t arithmetic to avoid FPU usage.

Note: Uses P1 meter's own epoch timestamp for peak timing (from OBIS 0-0:1.0.0).
