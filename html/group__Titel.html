‚úÖ Checklist: Nieuwe Module Documenteren

  1. Header file (bijvoorbeeld p_one.h):
    • @defgroup p_one P1 - API Referentie toegevoegd
    • @brief is √©√©n korte zin (oneliner) met punt aan einde
    • Verwijzing: "For complete documentation, see @ref p_one_doc"
    • @{ en @} om group te sluiten
  2. Module documentatie (p_one_doc.md):
    • Bestandsnaam = <module>_doc.md
    • Header: # <Module> - Documentatie {#module_doc} (met "- Documentatie" suffix!)
    • [TOC] toegevoegd na header
    • Navigatietabel met links naar:
      • API Referentie (@ref module)
      • Source files (@ref module.c)
      • Header files (@ref module.h)
      • Optioneel: Extra files (shell commands, etc.)
    • Overzicht sectie als start van inhoud
    • Inhoud volgens Documentation Guidelines
  3. Mainpage (docs/mainpage.md):
    • Module toegevoegd aan tabel met link @ref module_doc
    • Module naam in tabel is kort en duidelijk (zonder suffix)
  4. Verificatie:
    • Build: ./scripts/build-docs.sh --html --clean
    • Geen warnings over broken @ref links
    • Module verschijnt in "Modules" sectie
    • <module>_doc.md is zichtbaar onder module
    • Navigatietabel linkt correct naar API en source files
    • Bidirectionele navigatie werkt (mainpage ‚Üî DD ‚Üî API)

üö® Common Pitfalls

‚ùó Anchor Mismatch

Probleem: {#p_one_readme} maar bestand heet p_one_doc.md

Fix: Anchor moet matchen met bestandsnaam (zonder .md):

# P1 - Documentatie {#p_one_doc}

‚ùó Inconsistente Titel Formaten

Probleem: Titel zonder " - Documentatie" of " - API Referentie" suffix

Fix: Gebruik consistente titels:

  • ‚úÖ Design docs: # P1 - Documentatie {#p_one_doc}
  • ‚úÖ API reference: @defgroup p_one P1 - API Referentie
  • ‚ùó Oude stijl: # P1 Module {#p_one_doc} of @defgroup p_one P1 Module

‚ùó Broken Links

Probleem: ‘warning: unable to resolve reference to 'module_name’`

Oorzaak: @defgroup module_name ontbreekt of naam komt niet overeen.

Fix: Check dat @defgroup ID exact matcht met @ref naam:

// In header
@defgroup p_one P1 DSMR Module
// In p_one_doc.md
@ref p_one // ‚úÖ GOED
@ref P1 // ‚ùó FOUT

‚ùó Missing @ingroup

Probleem: <module>_doc.md verschijnt niet onder module in Doxygen.

Fix: Voeg @ingroup module_name toe aan doc bestand:

# P1 DSMR Module {#p_one_doc}
@ingroup p_one
