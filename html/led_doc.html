Overzicht

De led module biedt visuele feedback over de status van het apparaat door een RGB LED aan te sturen. Het maakt gebruik van een prioriteit-gebaseerd systeem om te bepalen welk LED-patroon (kleur en knipperfrequentie) wordt weergegeven. De status met de hoogste prioriteit overschrijft altijd statussen met een lagere prioriteit.

De module luistert naar ZBUS-berichten van de network, cloud, fota en trigger modules om de systeemstatus te bepalen.

Waarom? (Probleem & Motivatie)

Visuele Feedback voor IoT Device

Probleem: Embedded IoT devices hebben geen display. Gebruikers moeten kunnen zien:

  • Is het apparaat aan? (boot sequence)
  • Is het netwerk verbonden? (LTE-M/NB-IoT)
  • Is cloud connectie actief? (nRF Cloud)
  • Wordt er data verzameld? (P1 acquisitie)
  • Is er een error? (watchdog reset, modem failure)

Oplossing: RGB LED met verschillende kleuren en patronen - zie de volledige tabel hieronder in de "Wat?" sectie.

Non-Blocking LED Control

Design Rationale: LED animaties (blink, fade) moeten smooth zijn zonder main thread te blokkeren.

Oplossing:

  • PWM hardware voor color control (geen CPU overhead)
  • k_timer voor blink patronen (interrupt driven)
  • State based pattern selection (event driven updates)

Voordeel: LED updates kosten < 1% CPU, geen impact op critical timing.

ZBUS Interfaces

De led module is een pure "consumer" en luistert naar de volgende kanalen om de systeemstatus te bepalen:

  • Luistert naar:
    • NETWORK_CHAN: Ontvangt network_msg om de status van de modem en de LTE-verbinding te volgen.
    • CLOUD_CHAN: Ontvangt cloud_msg om de status van de nRF Cloud-verbinding te volgen.
    • FOTA_CHAN: Ontvangt fota_msg om FOTA-updates (downloaden, fouten) te detecteren.
    • TRIGGER_CHAN: Ontvangt trigger_msg om te weten wanneer data-acquisitie start.

Typische Scenario's

Normale Opstart (Happy Path):

  1. Power ON ‚Üí LED_STATUS_WAIT (Warm Wit Constant) - Modem initialiseert.
  2. Modem Ready ‚Üí LED_STATUS_CONNECTING (Warm Wit Knipperend) - Verbinden met LTE-netwerk.
  3. Netwerk Verbonden ‚Üí LED_STATUS_CLOUD_ERROR (Oranje Constant) - Cloud is nog niet gereed.
  4. Cloud Verbonden ‚Üí LED_STATUS_IDLE (Groen Langzaam Knipperend) - Systeem is operationeel.

Data Acquisitie:

  1. Systeem is IDLE ‚Üí LED_STATUS_IDLE (Groen Langzaam Knipperend).
  2. trigger_msg ontvangen ‚Üí LED_STATUS_ACQUIRING (Blauw Constant) - Wordt minimaal 2 seconden getoond.
  3. Acquisitie klaar ‚Üí Terug naar LED_STATUS_IDLE.

FOTA Update:

  1. Systeem is IDLE ‚Üí LED_STATUS_IDLE.
  2. fota_msg (DOWNLOADING) ontvangen ‚Üí LED_STATUS_FOTA (Warm Wit Snel Knipperend) - Dit heeft de hoogste prioriteit.
  3. Update voltooid/mislukt ‚Üí LED-patroon verandert naar FOTA_ERROR of terug naar IDLE na herstart.

Netwerkprobleem:

  1. Verbinding valt weg ‚Üí LED_STATUS_CONNECTING (Warm Wit Knipperend) - Probeert opnieuw te verbinden.
  2. Na 120 seconden geen succes ‚Üí LED_STATUS_NETWORK_ERROR (Rood Constant).
  3. Verbinding herstelt ‚Üí Gaat terug naar LED_STATUS_CLOUD_ERROR (Oranje) en vervolgens LED_STATUS_IDLE (Groen).

Hoe? (Architectuur & Gedrag)

Network State Machine

Voor het bepalen van de netwerkstatus (LED_STATUS_CONNECTING vs. LED_STATUS_NETWORK_ERROR) gebruikt de module een interne state machine. Dit voorkomt dat een normale (her)verbindingspoging onterecht als een fout wordt getoond.

+------------------+
| NEVER_CONNECTED |
+------------------+
| (modem init)
v
+------------------------------------------------------------------------+
| |
| +----------------+ (timeout > 120s) +-------------+ |
| | CONNECTING | -------------------------> | ERROR | |
| +----------------+ <------------------------- +-------------+ |
| ^ | (netwerk hersteld) ^ |
| | | (succes) | (disconnect) |
| (disconnect) | | |
| | v | |
| +-------------+ | |
| | CONNECTED | -------------------------------+ |
| +-------------+ |
| |
+------------------------------------------------------------------------+
  • NEVER_CONNECTED: Initiele staat. LED is WAIT (warm wit constant).
  • CONNECTING: Actief aan het verbinden. LED is CONNECTING (warm wit knipperend). Een timer van 120 seconden start.
  • CONNECTED: Verbinding is succesvol. De LED-status wordt bepaald door andere factoren (cloud, acquisitie, etc.).
  • ERROR: Als verbinden na 120 seconden niet lukt, gaat de state naar ERROR. LED is NETWORK_ERROR (rood constant). De state blijft ERROR totdat de verbinding herstelt.
