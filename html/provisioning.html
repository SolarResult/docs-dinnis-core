1.5 Certificaten Genereren

Genereer de CA certificaten voor SolarResult R&D devices:

create_ca_cert \
-c "NL" \
--st "Noord-Holland" \
-l "Alkmaar" \
-o "SolarResult B.V." \
--ou "RD" \
--cn "solarresult.nl" \
--dv 10950 \
--email "development@solarresult.nl" \
--path "utils/nrfcloud-certs/" \
--fileprefix "solarresult-rd-"

Parameters uitleg:

  • -c: Land (Country)
  • --st: Provincie/Staat (State)
  • -l: Stad (Locality)
  • -o: Organisatie
  • --ou: Organisatie Unit (bijv. R&D)
  • --cn: Common Name (domeinnaam)
  • --dv: Days Valid - 10950 dagen = 30 jaar geldigheid
  • --email: Contact email
  • --path: Output directory
  • --fileprefix: Prefix voor gegenereerde bestanden

Output:

  • solarresult-rd-XXXXX_ca.pem (CA certificaat)
  • solarresult-rd-XXXXX_prv.pem (Private key)

üîß Fase 2: Per Device Provisioneren

Voor elk individueel device moeten de volgende stappen doorlopen worden:

2.1 IMEI Programmeren

Stap 1: Flash PTI Modem Firmware

nrfutil device program --firmware utils/nrf-mfw/nrf91x1/mfw-pti_nrf91x1_2.3.8.zip

‚ö†Ô∏è Waarom PTI firmware? De PTI (Product Test Interface) versie stelt het programmeren van IMEI mogelijk.

Stap 2: Device Recoveren

nrfjprog --recover

Stap 3: Flash Modem Shell

west flash --build-dir build_mosh

Stap 4: IMEI Schrijven via RTT

Verbind met RTT en voer het volgende commando uit:

at 'AT%IMEIWRITE=0,"35829984xxxxxxc"'

of (alternatieve syntax):

at AT%IMEIWRITE=0,\"35829984xxxxxxc\"

‚úÖ Verificatie: Het device heeft nu een uniek IMEI.

2.2 Certificaten Installeren

Stap 1: Flash Productie Modem Firmware

nrfutil device program --firmware utils/nrf-mfw/nrf91x1/mfw_nrf91x1_2.0.3.zip

Stap 2: Certificaten Programmeren

device_credentials_installer \
--rtt \
--cmd-type "at" \
--term "CRLF" \
--dv 10950 \
--ca "utils/nrfcloud-certs/solarresult-rd-XXXXX_ca.pem" \
--ca-key "utils/nrfcloud-certs/solarresult-rd-XXXXX_prv.pem" \
--csv "utils/nrfcloud-onboard/solarresult-rd/onboard.csv" \
--id-str "solarresult-rd-" \
--id-imei \
--append \
--delete \
--devinfo "utils/nrfcloud-devinfo/solarresult-rd/devinfo.csv" \
--devinfo-append \
--replace \
--verify

‚ö†Ô∏è Let op: Controleer dat de juiste --ca en --ca-key paden gebruikt worden!

Parameters uitleg:

  • --rtt: Gebruik RTT voor communicatie met device
  • --cmd-type "at": Gebruik AT commando's
  • --term "CRLF": Line terminator (Carriage Return + Line Feed)
  • --dv 10950: Device certificaat geldig voor 30 jaar
  • --ca: Pad naar CA certificaat (gegenereerd in stap 1.5)
  • --ca-key: Pad naar CA private key
  • --csv: Output CSV met device credentials
  • --id-str: Prefix voor device ID
  • --id-imei: Gebruik IMEI als device ID
  • --append: Voeg toe aan bestaande CSV (indien aanwezig)
  • --delete: Verwijder oude certificaten van device
  • --devinfo: Output CSV met device informatie
  • --devinfo-append: Voeg toe aan bestaande devinfo CSV
  • --replace: Vervang bestaande entries in devinfo
  • --verify: Verifieer certificaten na installatie

Output files:

  • onboard.csv: Device credentials voor onboarding
  • devinfo.csv: Device informatie voor nRF Cloud
