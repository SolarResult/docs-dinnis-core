Overzicht

De Network module beheert de LTE-M/NB-IoT modem connectie voor de Nordic nRF9161 en implementeert een state machine voor robuuste netwerk connectiviteit. De module integreert met Zephyr's Connection Manager en publiceert network status updates via ZBUS.

Waarom? (Design Rationale)

Event-Driven Network Management

Probleem: Modem connectie is asynchroon en kan op elk moment wijzigen (signal loss, handover, operator switch).

Design Keuze: Event-driven architectuur met Zephyr NET Management callbacks.

Implementatie:

  • NET_EVENT_L4_CONNECTED â†’ Publiceert NETWORK_CONNECTED
  • NET_EVENT_L4_DISCONNECTED â†’ Publiceert NETWORK_DISCONNECTED
  • State machine reageert op ZBUS events (niet direct in callbacks)

Voordeel: Callbacks zijn lightweight (< 100Âµs), state transitions in thread context.

Modem Initialization Hook

Probleem: Modem library init is asynchroon - andere modules moeten wachten.

Design Keuze: NRF_MODEM_LIB_ON_INIT hook + semaphore synchronisatie.

Flow:

  1. Nordic modem library initialiseert (system startup)
  2. modem_lib_init_callback() wordt aangeroepen
  3. Semaphore wordt ge-released
  4. Network thread wacht op semaphore (met timeout)
  5. Publiceert NETWORK_MODEM_INIT zodra klaar

Voordeel: Gegarandeerde synchronisatie, timeout protection (120s default).

  3. Parent State: OPERATIONAL is parent voor beide child states, bevat common initialization logic.

Modem Initialization Details

Timeout Configuration:

  • Default: 120 seconds (CONFIG_APP_NETWORK_MODULE_MODEM_INIT_TIMEOUT_SECONDS)
  • Rationale: Modem kan lang duren bij eerste boot (SIM init, operator search)

Error Handling:

  • Timeout â†’ LOG_ERR + TODO: reboot request
  • Init failure â†’ LOG_ERR + TODO: reboot request

Sequence:

  1. Nordic modem library init (automatic via Zephyr sysinit)
  2. NRF_MODEM_LIB_ON_INIT hook triggered
  3. modem_lib_init_callback() called with result
  4. Semaphore released (k_sem_give)
  5. Network thread wakes up (k_sem_take succeeds)
  6. Publishes NETWORK_MODEM_INIT
