Energy Pulse Module

Overzicht

De Energy Pulse module monitort externe puls-tellers via GPIO interrupts en houdt persistente energie-tellers bij. Elke puls representeert 1 Wh (Watt-uur) energieverbruik of -productie.

Het systeem ondersteunt twee kanalen voor flexibele configuratie (bijv. één voor zonnepanelen opwek, één voor warmtepomp verbruik).

Waarom? (Probleem & Motivatie)

Persistente Energietellers

Probleem: Puls-tellers zoals S0-meters tellen alleen pulsen, niet absolute energie. Bij een reboot of stroomuitval zou de absolute energiestand verloren gaan.

Oplossing: De module gebruikt Zephyr Settings subsystem om de teller-standen persistent op te slaan in flash. Elke update wordt direct weggeschreven, waardoor geen data verloren gaat bij een reboot.

Use Case:

  • Warmtepomp verbruikt 5 kWh per dag
  • Bij stroomuitval om 14:00 (3 kWh al verbruikt)
  • Na herstart telt de meter verder vanaf 3 kWh
  • Geen data loss, accurate dagelijkse totalen

GPIO Interrupt Gedreven

Design Rationale: Puls-tellers genereren relatief weinig events (bijv. 1 Wh per puls = 8000 pulsen per dag bij 8 kWh verbruik). Polling zou inefficiënt zijn.

Oplossing:

  • GPIO interrupts met edge detection (rising edge)
  • Atomic counter increment in interrupt context
  • Minimal interrupt handler (< 1 µs)
  • Settings update via work queue (non-blocking)

Voordeel: CPU kan slapen tussen pulsen, optimale power efficiency.

Runtime Gedrag

Normal Operation:

t=0s: Boot → Load counters: CH1=1245 Wh, CH2=8732 Wh
t=45s: Pulse CH1 → Interrupt → counter++, save settings
t=122s: Pulse CH2 → Interrupt → counter++, save settings
t=600s: DATA_PUBLISH_REQUEST → Publish: {1246 Wh, 8733 Wh}
t=1200s: DATA_PUBLISH_REQUEST → Publish: {1247 Wh, 8735 Wh}

Reboot Scenario:

t=0s: Boot → Load: CH1=1245 Wh, CH2=8732 Wh
t=45s: Pulse CH1 → counter=1246 Wh → Save to flash
t=47s: REBOOT_REQUEST → Graceful shutdown
t=47.2s: System reboot
t=48s: Boot → Load: CH1=1246 Wh, CH2=8732 Wh ✅ No data loss!

Resultaten & Impact

Geheugen & Performance

  • RAM: ~400 bytes (atomic counters, work items, thread stack)
  • Flash: ~8 bytes persistent (2× uint32_t counters)
  • CPU overhead: < 1 µs per pulse (interrupt handler)
  • Settings write: ~5 ms per pulse (asynchronous, work queue)

Betrouwbaarheid

  • Data persistentie: 100% door flash opslag per puls
  • Reboot safe: Counters behouden bij power cycle
  • Interrupt safety: Atomic operations, geen race conditions

Toekomstige Uitbreidingen

  • ⏳ Configureerbare Wh per puls - Ondersteuning voor 10 Wh, 100 Wh meters
  • ⏳ Debouncing - Filter voor contact bounce (< 50 ms)
  • ⏳ Pulse validation - Max frequency check (detecteer hardware defect)
  • ⏳ Channel naming - Settings voor custom kanaal namen (i.p.v. CH1/CH2)
