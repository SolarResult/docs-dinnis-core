Detailed Description

Combined energy message structure.

Combines P1 DSMR meter data with pulse counter data into a single message. Used by Message Queue to buffer and send energy data to the cloud.

Both P1 and Pulse data are optional to handle cases where one source is temporarily unavailable or delayed.

Timestamps are stored as uptime_ms during data collection and buffering. Conversion to Unix epoch happens only when sending to cloud (after time sync).
