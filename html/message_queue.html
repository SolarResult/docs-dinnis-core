Message Queue Module

Overzicht

De Message Queue module combineert energie metingen van P1 (DSMR slimme meter) en Pulse (puls-tellers) modules, buffert deze voor cloud verzending, en implementeert een state machine voor robuuste delivery met retry logica.

De module gebruikt een integer-only data pipeline voor optimaal geheugengebruik en elimineert alle FPU overhead.

Waarom? (Probleem & Motivatie)

RAM Optimalisatie: Integer-Only Pipeline

Oorspronkelijk probleem: 94.3% RAM gebruik (199.4 KB) - kritiek niveau!

Analyse: De module combineerde P1 en Pulse data en bufferde deze als double (8 bytes per veld):

Old struct combined_energy_msg:
- 8× double energie velden (8 bytes each) = 64 bytes
- 4× double vermogen velden (8 bytes each) = 32 bytes
- Timestamps, metadata = 56 bytes
Total: ~152 bytes per bericht
Queue: 400 berichten × 152 bytes = ~61 KB RAM

Ontdekking: OBIS parser gebruikt al int32_t fixed-point intern!

Oude inefficiënte flow:
OBIS parser (int32_t) → double conversie (FPU!)
→ opslag (8 bytes) → int32_t conversie (FPU!)

Oplossing: Direct int32_t usage:

New struct combined_energy_msg:
- 8× int32_t energie velden (4 bytes each) = 32 bytes
- 4× int32_t vermogen velden (4 bytes each) = 16 bytes
- Timestamps, metadata = 56 bytes
Total: ~104 bytes per bericht
Queue: 400 berichten × 104 bytes = ~42 KB RAM

Result: 19 KB RAM bespaard alleen in queue! (31% reductie)

Totale Impact RAM Optimalisatie

Na integer-only pipeline in P1, Pulse en Message Queue modules:

  • Voor: 199.4 KB (94.3% usage)
  • Na: 154.6 KB (73.08% usage)
  • Besparing: 44.8 KB (22.4% reductie)

Veiligheidsmarge: Van kritieke 5.7% naar gezonde 27% vrij!

Delivery Garanties

Probleem: Netwerk kan instabiel zijn (LTE-M cell handover, zwak signaal)

Scenario: 3 berichten klaar, netwerk weg
❌ Zonder buffering: Data verloren
✅ Met queue: Berichten blijven in buffer tot netwerk terug

Oplossing: State machine met states:

  • COLLECTING_DATA - Verzamel P1 + Pulse data
  • OPERATIONAL - Cloud connected, verzend als queue > threshold
  • WAITING_FOR_NETWORK - Buffer data tot netwerk terug
  • WAITING_FOR_CLOUD - Buffer data tot cloud connection

Queue Management

Ring Buffer Implementatie:

#define QUEUE_CAPACITY 400
static struct combined_energy_msg queue[QUEUE_CAPACITY];
static size_t queue_head = 0; // Write index
static size_t queue_tail = 0; // Read index
static size_t queue_count = 0; // Items in queue

Enqueue (bij P1/Pulse data):

1. Check queue not full (count < capacity)
2. Write to queue[head]
3. head = (head + 1) % capacity (wrap around)
4. count++

Dequeue (bij cloud verzending):

1. Check queue not empty (count > 0)
2. Read from queue[tail]
3. tail = (tail + 1) % capacity
4. count--

Overflow gedrag: Als queue vol (400 berichten):

  • ❌ Oude implementatie: Crash of data loss
  • ✅ Nieuwe implementatie: Log WARNING, drop oldest (FIFO)

Verzend Logica

/* OPERATIONAL state - verzend als threshold bereikt */
if (queue_count >= CONFIG_MESSAGE_QUEUE_SEND_THRESHOLD) {
// Batch send: stuur meerdere berichten tegelijk
int sent = 0;
while (queue_count > 0 && sent < MAX_BATCH_SIZE) {
send_to_cloud(&queue[tail]);
dequeue();
sent++;
}
}

Configuratie:

  • SEND_THRESHOLD = 3 berichten (default)
  • MAX_BATCH_SIZE = 10 berichten per verzend cyclus
  • Balans tussen latency en efficiency

Betrouwbaarheid

Queue overflow test (stress test):

Scenario: Network down voor 72 uur (3 dagen)
- Publish rate: 2 berichten/10 min = 12 berichten/uur
- Total: 72h × 12 = 864 berichten
- Queue capacity: 400 berichten
Result:
- Eerste 400 berichten: Gebufferd ✅
- Berichten 401-864: Gedropt met LOG_WRN ⚠️
- Na network recovery: 400 berichten verzonden ✅

Praktijk:

  • Typisch network outage: < 2 uur
  • Max gebufferd: 24 berichten
  • Queue overflow: Zeer zeldzaam (< 0.1% tijd)

Toekomstige Uitbreidingen

  • ⏳ Persistent queue - Flash backed queue voor reboot safety
  • ⏳ Compression - LZ4 compressie voor cloud payload (extra 30-40% besparing)
  • ⏳ Priority levels - Kritieke berichten altijd versturen
  • ⏳ Adaptive batching - Batch size aanpassen op basis van queue depth
