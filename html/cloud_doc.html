2. Overzicht

De Cloud module is het hart van de dataverwerking en communicatie. Het is een robuuste, state-machine-gedreven module die verantwoordelijk is voor:

  • Connectiviteit: Het beheren van een persistente verbinding met nRF Cloud, inclusief automatische herverbindingspogingen met exponentiÃ«le backoff.
  • Data Acquisitie & Buffering: Het luisteren naar data van de P1 en Pulse modules en deze bufferen in een interne message queue.
  • Data Verzending: Het periodiek versturen van de gebufferde data naar nRF Cloud.
  • FOTA Updates: Het orkestreren van Firmware-Over-The-Air updates, inclusief de cruciale "boot confirmation" om rollbacks te voorkomen.

States:

  1. STATE_WAITING_FOR_MODEM: De initiÃ«le staat na het opstarten. Wacht tot de Network module de modem heeft geÃ¯nitialiseerd.
  2. STATE_DISCONNECTED: De modem is klaar, maar er is nog geen actieve netwerkverbinding. Wacht op een network_msg die aangeeft dat het netwerk verbonden is.
  3. STATE_CONNECTING: Probeert actief verbinding te maken met nRF Cloud. Gebruikt een exponentieel backoff-mechanisme bij mislukte pogingen om het netwerk niet te overbelasten.
  4. STATE_CLOUD_READY: Succesvol verbonden met nRF Cloud. In deze staat kan het apparaat data verzenden en ontvangen. De module voert hier ook de FOTA boot confirmation uit. Bij een disconnect-event of netwerkverlies valt de state machine terug naar DISCONNECTED of CONNECTING.

3.2 Data Acquisitie en Buffering

Een kernfunctionaliteit van de Cloud module is het verzamelen en bufferen van data.

  1. Luisteren: De module luistert naar de p1_chan en pulse_chan.
  2. Buffering: Wanneer een bericht van de P1 of Pulse module wordt ontvangen, wordt dit bericht gekopieerd naar een interne message queue (cloud_msgq). Deze queue fungeert als een FIFO-buffer.
  3. Verzending: Periodiek, getriggerd door de Timing module, verwerkt de Cloud module de berichten in de queue en verstuurt ze naar nRF Cloud.

Dit ontwerp ontkoppelt de data-acquisitie van de data-verzending, wat het systeem robuust maakt tegen tijdelijk verlies van de cloudverbinding.

3.3 FOTA (Firmware-Over-The-Air)

De Cloud module orkestreert het FOTA-proces:

  1. Detectie: De nRF Cloud bibliotheek detecteert een nieuwe FOTA-taak.
  2. Download: De bibliotheek start de download. De Cloud module publiceert de voortgang op de fota_chan, waardoor de LED module visuele feedback kan geven.
  3. Voltooiing: Na een succesvolle download publiceert de Cloud module een REBOOT_TYPE_FOTA bericht op de reboot_chan.
  4. Herstart: De Reboot module zorgt voor een graceful shutdown, waarna MCUboot de firmware-update uitvoert.
  5. Boot Confirmation: Na de herstart, in de CLOUD_READY state, roept de Cloud module boot_write_img_confirmed() aan. Dit is een cruciale stap die aan MCUboot signaleert dat de nieuwe firmware succesvol is opgestart, en voorkomt dat MCUboot na een paar mislukte pogingen terugvalt op de oude firmware.

4. ZBUS Interface

4.1 Geobserveerde Kanalen

Kanaal Type Afzender Doel
timing_chan timing_msg Timing Start het periodiek versturen van gebufferde data.
p1_chan p1_msg P1 Ontvangt P1-data om te bufferen in de message queue.
pulse_chan pulse_msg Pulse Ontvangt puls-data om te bufferen in de message queue.
network_chan network_msg Network Reageert op veranderingen in de netwerkstatus voor de state machine.
reboot_chan reboot_msg Reboot Start een graceful disconnect bij een shutdown-verzoek.

4.2 Gepubliceerde Kanalen

Kanaal Type Doel
cloud_chan cloud_msg Publiceert de status van de nRF Cloud verbinding (CLOUD_READY / CLOUD_NOT_READY).
fota_chan fota_msg Publiceert de status en voortgang van FOTA-updates.
